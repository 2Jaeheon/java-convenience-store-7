# 요구사항 정리

목표: 할인 혜택과 재고 상황에 따른 최종 결제 금액을 계산하고 안내하는 결제 시스템

사용자가 입력한 상품의 가격과 수량을 통해 최종 결제 금액 계산

- 총 구매액은 상품별 가격과 수량을 곱하여 계산, 프로모션 및 멤버십 할인 정책을 반영 -> 최총 결제 금액 산출

## 재고 관리

- 재고 수량을 고려해서 결제 가능 여부를 파악
- 구매하면 상품의 재고에서 차감

## 프로모션 할인

- 오늘 날짜가 프로모션 기간 안에 있을 때 할인 적용
- 프로모션은 N개 구매시 1개 무료증정 형태
- 1+1또는 2+1이 지정된 상품에 적용
- 프로모션 혜택은 프로모션 재고 내에서만 적용 (일반 재고에는 적용 X)
- 프로모션 재고가 부족하면 일반 재고를 사용함
- 만일 프로모션 재고가 부족하는 경우에는 일부 수량을 정가로 결제해야함. -> 안내
- 프로모션 적용이 가능한데 적게 가져오면 추가로 가져오면 혜택 받을 수 있는 것 안내

## 멤버십 할인

- 멤버십 할인은 프로모션 미적용 금액의 30% 할인
- (예를 들어서 2+1이나 1+1 된 상품은 멤버십 할인에서 제외되는 것)
- 프로모션 적용 후 남은 금액에 대해서 멤버십 할인을 적용
- 최대 8000원까지 할인 가능

## 영수증

영수증에는 다음 항목이 존재해야함

- 구매 상품 내역: 구매한 상품명, 수량, 가격
- 증정 상품 내역: 프로모션에 따라 무료로 제공된 증정 상품의 목록
- 금액 정보
    - 총구매액: 구매한 상품의 총 수량과 총 금액
    - 행사할인: 프로모션에 의해 할인된 금액
    - 멤버십할인: 멤버십에 의해 추가로 할인된 금액
    - 내실돈: 최종 결제 금액
